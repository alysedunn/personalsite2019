How to Deploy your Python Flask app to AWS Elastic Beanstalk

10Aug- 2015
424 Shares

Three years ago, I was a new developer trying (and failing) to deploy my simple Python flask application to Amazon Web Services' (AWS) Elastic Beanstalk (EB). In the process, I stumbled across this documentation from flask, this documentation from AWS, and this blog post on how to use them together, all of which were extremely helpful. However, I was so new to development that even with these and other helpful online resources, I found the entire process confusing, and I struggled to debug the deployment issues that arose during the process. I still run into some of those deployment issues when I deploy flask apps to EB today, so I've decided not to let another year go by without creating a detailed tutorial on deploying flask apps to EB, complete with examples of common errors that pop up during the process, and how to debug those errors. So, with that, let's get push get that flask app our of your head and into production!


Step 1: Make a flask app!
If you're a true beginner, just getting a flask app up and running locally might be a significant feat, much less deploying it to production via EB. However, there are lots of truly excellent tutorials online to guide you that are appropriate for even novice programmers, like this one, which is a part of the official flask documentation. Since the focus of this post is on deploying flask apps that you've already created, I won't provide a tutorial on creating the apps themselves, but regardless of which tutorial you use and what kind of app you create, be sure that your app meets the criteria below. If it doesn't, change it before continuing with this tutorial. If you do not follow the instructions below exactly, I gaurantee that you will encounter errors that I won't even begin to cover here.

  1. Your 'application' file, i.e. the the file that contains your main python code (it probably has a line with from flask import Flask at the top of the file), must be called application.py.
  2. Within your application file (which should be named application.py), ensure that the variable that you assign to Flask(__name__) is application, not app. So, you should have a line in a file named application.py with application = Flask(__name__).
  3. At the end of your application file, before to set the "__name__" attribute equal to "__main__". So, you should have a small clause at the end of the file that looks something (or exactly) like this:
    if __name__ == "__main__":
        application.debug = False
        application.run()


Step 2: Make an AWS Account!
Here's the link to visit to create an AWS account. Two notes about setting up an account:

  1. If you'd like an overview of all of the steps involved in creating an account, this outlines that process.
  2. You must enter a credit card in order to sign up for an AWS account. How much will you get charged on your credit card when you register? Well that depends on how much you plan on utilizing AWS, but for context, this blog is a flask app that I run on EB, and AWS bills me about $20 per month to keep it up and running.
