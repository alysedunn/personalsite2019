How to Deploy your Python Flask app to AWS Elastic Beanstalk

1Feb- 2019

Three years ago, I was a new developer trying (and failing) to deploy my simple Python flask application to Amazon Web Services' (AWS) Elastic Beanstalk (EB). In the process, I reviewed this documentation from Flask, this documentation from AWS, and this blog post on using the two together. All were helpful, but I was so new to development that even with these and other resources, I found the entire process terribly frustrating as I ran into one bug after another and tried to Google my way through each one of them. I still run into some of those bugs when I deploy my Flask apps to AWS today, but fortunately after years of seeing them, I can power through them a lot more quickly than I used to. In an effort to continue making this process as seamless as possible for myself, and to assist others who may be struggling, I present the following step-by-step guide to deploying your own flask app to AWS Elastic Beanstalk. The example assumes that you are using Python 2.7. Note for those new to Python, Flask, AWS, or all three: Be sure to click on the links and icons throughout this post for more detailed information on concepts, terms, and tools that may be new to you.


Step 1: Create a virtual environment!
To be fair, you can get a Flask app up and running successfully without the use of a virtual environment. You might even be able to get that app deployed to EB without using a virtual environment. However, I ran into various errors in step 4 below when I did notuse a virtual environment. So, if you want this deployment to go smoothly, I recommend starting here. To create a virtual environment, you must first install virtualenv by running be installed by running pip install virtualenv.


Step 2: Make a flask app!
If you're a true beginner, getting a flask app up and running locally might be a feat in and of itself. Fortunately, there are many great tutorials that can help, like this one, from the official creators of flask. Since the focus of this post is on deploying flask apps that you've already created, I won't provide a tutorial on creating the apps themselves. I'll leave the actual app creation to you. However, I will provide the following tips to ensure that your app includes features that will ensure ensure a smooth deployment to AWS EB, namely:

  1. Name your 'application' file application.py. Not app.py or taco.py or anything else that's not application.py. Your application file is the file that contains your main python code. You likely have from flask import Flask towards the top of the file).
  2. In your application file, ensure that the value of Flask(__name__) is application, not app or taco or anything else that's not application.
  3. In your application file, ensure that the value of "__name__" is "__main__", not "__app__" or "__taco__" or anything else that's not "__main__". So, at the end of your application file, you should have a short function that looks something like this:
    if __name__ == "__main__":
        application.debug = False
        application.run()


Step 3: Make an AWS Account!
To create an account in AWS, start here, and simply follow the prompts. It takes less than 5 minutes to create a basic account with a login and password, but after you create the account, you'll also want to do the following:

  1. If you'd like an overview of all of the steps involved in creating an account, this outlines that process.
  2. You must enter a credit card in order to sign up for an AWS account. How much will you get charged on your credit card when you register? Well that depends on how much you plan on utilizing AWS, but for context, this blog is a flask app that I run on EB, and AWS bills me about $20 per month to keep it up and running.
